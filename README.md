# 1. Использованные виджеты:
## Основные виджеты компоновки:

Column – для вертикального расположения элементов

Row – для горизонтального расположения иконок и текста в кнопках

Card – для стильного отображения значения счётчика

Container – для создания стилизованных блоков с фоном и границами

Padding – для внутренних отступов

SizedBox – для фиксированных промежутков между элементами

## Контейнеры и вспомогательные виджеты:

AppBar – верхняя панель с названием приложения

Scaffold – базовая структура экрана

Icon – для добавления иконок

Text – для отображения текста различного формата

## Элементы управления:

ElevatedButton – основные кнопки "Увеличить" и "Сбросить"

GestureDetector (встроен в кнопки) – для обработки нажатий

# 2. Реализация управления состоянием:
## Ключевые компоненты:

StatefulWidget (CounterApp) – основной виджет с изменяемым состоянием

Состояние (_CounterAppState) – содержит переменную counter и методы для её изменения

setState() – метод, который уведомляет Flutter о необходимости перерисовки интерфейса

## Логика обновления:

dart
void incrementCounter() {
  setState(() {
    counter++;  // Изменение данных
  });  // После этого Flutter автоматически перерисовывает интерфейс
}
## Принцип работы:

Пользователь нажимает кнопку

Вызывается метод (например, incrementCounter())

Внутри setState() изменяется значение counter

Flutter сравнивает новое состояние с предыдущим

Перерисовываются только те виджеты, которые зависят от изменившихся данных

# 3. Обрабатываемые события:
## 1. Короткое нажатие (onPressed):

Кнопка "Увеличить": counter++ (увеличение на 1)

Кнопка "Сбросить": counter = 0 (обнуление счётчика)

## 2. Долгое нажатие (onLongPress):

Кнопка "Увеличить": counter += 10 (увеличение на 10)

## 3. Условное отображение:

Текст меняется в зависимости от значения счётчика:

counter == 0: "Счётчик сброшен"

counter > 10: "Отлично! Значение > 10!"

Иначе: "Продолжайте увеличивать"

# 4. Особенности реализации:
## Стилизация:

Разные цвета контейнеров для кнопок (синий для увеличения, красный для сброса)

Тени и скруглённые углы для Card и контейнеров

Адаптивные отступы через Padding

Изменение цвета текста при достижении определённых значений

## Архитектура:

Чёткое разделение логики (методы изменения состояния) и представления (виджеты)

Использование констант для единообразия стилей

Декларативный подход: интерфейс полностью описывает своё состояние в любой момент

## Дополнительные функции:

Информационная панель с текущей статистикой

Визуальная обратная связь при взаимодействии

Адаптивный текст, который меняется в зависимости от состояния
